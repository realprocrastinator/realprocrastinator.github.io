Sorting Algorithms Review

In this chapter, we will cover ten commonly seen sorting algorithms at the moment. We will describe the algorithm, look at the pseudocode and implement it in C/C++, Python. And finally, we will do a unit test and benchmark on each of them as well. Micro-benchmark will show the comparison between different algorithms. And the a Macro-benchmark mark will compare the same algorithm implemented in various languages.

Common

The following table summarizes the running times of the common sorting algorithm. As usual, n denotes the number of items to sort. For counting sort, the items to sort are integers in the set {0, 1......, k}. For radix sort, each item is a d -digit number, where each digit takes on k possible values. For bucket sort, we assume that the keys are real numbers uniformly distributed in the half-open interval [0, 1).

Algorithm Worst-case running time Average-case/expected running time
Bubble Sort Θ (n2) Θ (n2)
Insertion Sort Θ (n2) Θ (n2)
Merge Sort Θ (n lg n) Θ (n2)
Quick Sort Θ (n2) Θ (n lg n)
Heap Sort Θ (n lg n) —
Counting Sort Θ (k + n) Θ (k + n)
Radix Sort Θ (d(k + n)) Θ (d(k + n))
Bucket Sort Θ (n2) Θ (n)   (expected)
Selection Sort Θ (n2) Θ (n2)
Shell Sort Θ (n2) Θ (n2)
